### Soft Fork (блок #5)

- **Повышение сложности**: `00` → `000`
- **Новое поле**: `Teacher`
- **Обратная совместимость**: старые блоки (0-4) остаются валидными, новое поле опциональное

### Hard Fork (блок #10)

- **Смена алгоритма**: SHA3-384 → SHA3-512 (96 символов → 128 символов в хэше)
- **Смена формата**: JSON → CSV
- **НЕ совместим**: создаётся отдельная ветка, несовместимая с оригиналом

## Ключевые моменты

### 1. Soft Fork - Обратная совместимость

- Блоки 0-4 валидны БЕЗ поля Teacher
- Блоки 5+ могут иметь пустое Teacher (предупреждение, но не ошибка)
- Сложность майнинга растёт, но старые блоки не перемайниваются

### 2. Hard Fork - Разделение цепи

- Два файла: `blockchain.json` и `blockchain.csv`
- Разные алгоритмы хэширования (SHA3-384 vs SHA3-512)
- Разные форматы хранения (JSON vs CSV)
- Невозможно валидировать блоки одной ветки правилами другой

### 3. Майнинг

- Блоки 0-4: майнятся быстро (~0-50ms)
- Блоки 5-9: майнятся медленнее (~100-500ms) из-за сложности 000
- Блоки 10+: майнятся медленнее + используют SHA3-512
